# 数据管理可视化系统 - 功能说明文档

## 一、系统概述

数据管理可视化系统是一个**动态数据管理平台**，用户通过可视化界面创建数据表后，系统自动生成对应的管理页面，实现零代码数据管理。同时集成AI对话系统，提供智能化的数据管理辅助能力。

### 核心特性
- 📋 **动态建表** — 可视化定义数据表结构，无需SQL
- 🎯 **页面自动生成** — 建表后自动生成CRUD管理页面
- 📊 **数据可视化** — 多种图表展示数据
- 🤖 **AI辅助** — 大模型对话，智能管理数据

---

## 二、功能模块

### 2.1 数据表管理模块

| 功能 | 说明 |
|------|------|
| 创建数据表 | 可视化定义表名、字段名、字段类型、约束条件 |
| 编辑表结构 | 修改字段、添加/删除字段、调整字段属性 |
| 删除数据表 | 删除表及关联数据（需确认） |
| 表模板管理 | 保存常用表结构为模板，快速复用 |

**支持的字段类型**：
- 基础类型：文本、数字、布尔值、日期
- 高级类型：单选、多选、富文本、图片、文件
- 关联类型：关联其他表、多对多关系

---

### 2.2 自动页面生成模块

建表完成后，系统自动生成以下管理页面：

| 页面 | 功能 |
|------|------|
| 数据列表页 | 表格展示、分页、排序、筛选、搜索 |
| 数据新增页 | 根据字段类型自动生成表单 |
| 数据编辑页 | 复用新增表单，预填充数据 |
| 数据详情页 | 字段值展示、关联数据跳转 |
| 批量操作 | 批量删除、批量修改、导入导出 |

**智能表单生成规则**：
- 文本类型 → 输入框（自动判断长度，长文本用文本域）
- 数字类型 → 数字输入框（支持范围限制）
- 单选类型 → 下拉选择 / 单选按钮
- 多选类型 → 多选框 / 标签选择器
- 日期类型 → 日期选择器
- 关联类型 → 关联选择器（支持搜索、创建）
- 图片类型 → 图片上传组件
- 富文本类型 → 富文本编辑器

### 2.4 AI模型管理模块

系统支持多模型配置，用户可通过界面管理多个AI模型：

| 功能 | 说明 |
|------|------|
| 添加模型 | 配置模型名称、类型、API地址、密钥等 |
| 编辑模型 | 修改模型配置信息 |
| 删除模型 | 移除不需要的模型配置 |
| 启用/禁用 | 控制模型是否可用 |
| 设置默认 | 指定默认使用的模型 |
| 测试连接 | 验证模型配置是否正确 |

**支持的模型类型**：
- OpenAI（GPT-3.5、GPT-4等）
- 国产大模型（通义千问、文心一言、讯飞星火、智谱GLM等）
- 本地模型（Ollama、LocalAI等）
- 自定义API（兼容OpenAI格式的接口）

**模型配置项**：
| 配置项 | 说明 |
|--------|------|
| 模型名称 | 自义显示名称，如"通义千问-主力" |
| 模型类型 | openai / qwen / wenxin / custom 等 |
| API地址 | 模型接口地址 |
| API密钥 | 认证密钥（加密存储） |
| 模型标识 | 具体模型ID，如 gpt-4、qwen-turbo |
| 参数配置 | temperature、max_tokens等 |
| 状态 | 启用/禁用 |
| 是否默认 | 默认使用的模型 |

| 功能 | 说明 | 示例指令 |
|------|------|---------|
| 模型切换 | 对话时可选择不同模型 | 选择"通义千问"进行对话 |
| 数据查询 | 自然语言查询数据 | "查询所有金丹境界的人物" |
| 数据分析 | 智能分析数据规律 | "分析各宗门的实力分布" |

| 数据建议 | 数据管理建议 | "我的数据表结构合理吗" |
| 关系梳理 | 梳理数据关联关系 | "梳理主角的人际关系" |
| 内容补全 | 补充缺失数据 | "为这些人物生成背景故事" |
| 对话历史 | 保存对话记录 | 查看历史对话内容 |
| Token统计 | 记录每次消耗的token数 | 查看各模型消耗统计 |

---

### 2.6 Token消耗统计模块

系统自动记录每次AI对话的token消耗：

| 功能 | 说明 |
|------|------|
| 自动记录 | 每次对话自动记录输入/输出token数 |
| 分模型统计 | 按模型统计总消耗 |
| 分时段统计 | 按日/周/月统计消耗趋势 |
| 费用估算 | 根据模型定价估算费用 |
| 消耗预警 | 设置阈值，超限提醒 |

**统计维度**：
- 总消耗量（累计token）
- 输入token / 输出token
- 按模型分组统计
- 按日期范围筛选
- 消耗趋势图表

## 四、使用流程

### 4.1 创建数据表

```
1. 进入「表管理」页面
2. 点击「新建数据表」
3. 填写表名称、描述
4. 添加字段：
   - 字段名称：姓名
   - 字段类型：文本
   - 是否必填：是
5. 继续添加其他字段...
6. 保存表结构
7. 系统自动生成管理页面
```

### 4.2 管理数据

```
1. 进入「数据管理」页面
2. 选择已创建的数据表
3. 系统展示该表的管理界面
4. 进行增删改查操作
```

### 4.3 可视化展示

```
1. 进入「数据可视化」页面
2. 选择数据表和展示方式
3. 配置图表参数
4. 保存视图配置
```

### 4.4 AI模型配置

```
1. 进入「AI模型管理」页面
2. 点击「添加模型」
3. 填写配置信息：
   - 模型名称：通义千问-主力
   - 模型类型：qwen
   - API地址：https://dashscope.aliyuncs.com/api/v1
   - API密钥：sk-xxx
   - 模型标识：qwen-turbo
   - 参数配置：temperature=0.7, max_tokens=2000
4. 点击「测试连接」验证配置
5. 保存并启用模型
6. 设置为默认模型（可选）
```

### 4.5 AI对话使用

```
1. 进入「AI助手」页面
2. 选择使用的模型（默认自动选中）
3. 输入自然语言指令
4. AI解析意图并执行
5. 返回结果或建议
6. 可查看历史对话记录
```

### 4.6 Token消耗查看

```
1. 进入「Token统计」页面
2. 选择统计维度：
   - 按模型筛选
   - 按时间范围筛选
3. 查看统计数据：
   - 总消耗量
   - 输入/输出token分布
   - 费用估算
4. 查看趋势图表
5. 导出统计报表
```

## 七、技术选型建议

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | Vue 3 + TypeScript | 响应式框架，组件化开发，国内生态完善 |
| **UI组件库** | TDesign | 腾讯企业级组件库，表单/表格组件丰富，支持动态渲染 |
| **可视化图表** | ECharts | 百度开源图表库，图表类型全面，中文文档完善 |
| **后端框架** | NestJS (Node.js) | TypeScript全栈统一，模块化架构，便于动态表结构处理 |
| **数据库** | PostgreSQL | 支持JSON字段存储复杂配置，动态建表灵活，性能优秀 |
| **AI集成** | LangChain.js | 多模型统一适配，支持OpenAI/通义千问/文心一言等主流模型 |
