## 产品概述

数据管理可视化系统是一个动态数据管理平台，用户通过可视化界面创建数据表后，系统自动生成对应的管理页面，实现零代码数据管理。同时集成AI对话系统，提供智能化的数据管理辅助能力。

## 核心功能

- 动态建表：可视化定义数据表结构，支持多种字段类型（文本、数字、布尔、日期、单选、多选、富文本、图片、关联等）
- 页面自动生成：建表后自动生成CRUD管理页面，包括数据列表、新增、编辑、详情页
- 数据可视化：表格、卡片、饼图、柱状图、折线图、雷达图、关系图等多种图表展示
- AI模型管理：多模型配置管理，支持OpenAI、通义千问、文心一言等主流模型
- AI对话辅助：自然语言查询数据、数据分析、数据生成、关系梳理等智能功能
- Token消耗统计：自动记录消耗、分模型统计、费用估算、消耗预警

## 技术栈选型

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端框架 | Vue 3 + TypeScript | 响应式框架，组件化开发，国内生态完善 |
| UI组件库 | TDesign | 腾讯企业级组件库，表单/表格组件丰富，支持动态渲染 |
| 可视化图表 | ECharts | 百度开源图表库，图表类型全面，中文文档完善 |
| 后端框架 | NestJS (Node.js) | TypeScript全栈统一，模块化架构，便于动态表结构处理 |
| 数据库 | PostgreSQL | 支持JSON字段存储复杂配置，动态建表灵活，性能优秀 |
| AI集成 | LangChain.js | 多模型统一适配，支持OpenAI/通义千问/文心一言等主流模型 |


## 实现方案

### 系统架构

采用前后端分离架构，三层架构设计：

```
前端展示层(Vue3) <---> 业务逻辑层(NestJS) <---> 数据存储层(PostgreSQL)
```

### 核心技术实现

**动态建表引擎**

- 元数据驱动：通过sys_table和sys_field元数据表管理表结构定义
- 动态SQL：运行时根据元数据生成DDL语句创建实际数据表
- Schema缓存：缓存表结构信息，提升动态查询性能

**页面自动生成器**

- 字段类型映射：根据字段类型自动选择对应的表单组件
- 动态路由：根据表定义动态注册前端路由
- 配置驱动：表单验证规则、显示字段均由配置决定

**AI引擎集成**

- LangChain.js适配：统一封装不同AI模型的调用接口
- 上下文管理：维护对话上下文，支持多轮对话
- Token追踪：拦截API响应，自动记录消耗数据

## 页面规划

### 1. 数据表管理页

- **顶部导航栏**：系统Logo、全局搜索、用户信息
- **侧边菜单栏**：功能模块导航，支持折叠
- **内容区域**：
- 表格列表区：展示所有数据表，支持搜索、筛选
- 操作按钮区：新建表、导入模板
- 表结构编辑器：可视化字段配置面板

### 2. 数据管理页(动态CRUD)

- **数据列表区**：动态表格展示，支持分页、排序、筛选
- **操作工具栏**：新增、批量删除、导入导出按钮
- **侧滑详情面板**：点击行展示详情信息
- **弹窗表单**：新增/编辑数据的动态表单

### 3. 数据可视化页

- **数据源选择**：选择要展示的数据表
- **图表类型选择器**：选择图表展示方式
- **图表配置面板**：配置X/Y轴、分组、筛选条件
- **图表展示区**：ECharts图表渲染区域

### 4. AI模型管理页

- **模型列表**：卡片式展示已配置的模型
- **配置表单**：模型名称、类型、API地址、密钥等配置项
- **连接测试**：测试按钮与测试结果展示
- **状态指示**：启用/禁用开关、默认模型标记

### 5. AI对话页

- **模型选择器**：顶部下拉选择当前使用的模型
- **对话历史**：左侧会话列表
- **对话区域**：中间消息展示区，支持Markdown渲染
- **输入区域**：底部输入框，支持多行输入

### 6. Token统计页

- **统计概览卡片**：总消耗、本月消耗、预估费用
- **趋势图表**：折线图展示消耗趋势
- **消耗明细表**：按模型/按日期分组的消耗明细
